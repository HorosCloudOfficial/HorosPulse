Cursor IDE Shortcuts Windows

Wichtige KI-Workflow-Tastenkürzel für Cursor IDE auf Windows — Agent, Chat, Inline Edit, Apply/Reject, Modell- & Modusauswahl, Kontext-Eingabe. Kein Code-Navigation (F12, Rename, Multi-Cursor).

docs/screenshots/features/cursor-shortcuts/01-agent-panel-diff.png

🤖 Die drei Interaktionsmodi

Cursor IDE bietet drei distinkte KI-Modi. Der Modus bestimmt, wie tief der Agent in den Code eingreift.

🤖

Agent

Autonome, mehrstufige Aufgaben. Liest und schreibt Dateien, führt Terminal-Befehle aus, schlägt Diffs vor.

Öffnen: Ctrl+I
💬

Chat / Ask

Konversationell, schreibgeschützt. Erklärt Code, beantwortet Fragen ohne direkte Datei-Änderungen.

Öffnen: Ctrl+L
✏️

Inline Edit

Direkt im Editor. Selektierter Block wird als Kontext übergeben; Diff erscheint sofort in der Datei.

Öffnen: Ctrl+K
Modus wechseln: Ctrl+. öffnet das Auswahlmenü. Shift+Tab rotiert Agent → Ask → Manual.

❓ Häufige Fragen & Änderungen

Cursor 0.45+: Review-Häkchen verschwunden

Der separate „Review"-Checkpoint vor dem Übernehmen von Änderungen wurde in Cursor 0.45 entfernt. Änderungen erscheinen direkt als Diff; Ctrl+Enter = Accept, Ctrl+Backspace = Reject. Einzelne Diff-Hunks können weiterhin per Gutter-Klick akzeptiert oder abgelehnt werden.

Ctrl+/ kommentiert nicht mehr aus

Ctrl+/ wechselt im Panel-Kontext das Modell. Im Editor (ohne Panel-Fokus) funktioniert es weiterhin als „Zeile auskommentieren". Zur Trennung: keybindings.json mit "when": "editorTextFocus && !inCursorChat" nutzen. Details: 06-faq-troubleshooting.md.

Ctrl+Shift+J vs. Ctrl+,

Ctrl+, öffnet VS-Code-Settings (Editor, Extensions, Theme). Ctrl+Shift+J öffnet Cursor-Settings (KI-Modelle, Privacy, Rules, Beta). Beide UIs sind separat — Cursor-spezifische Einstellungen sind nur in Cursor-Settings zu finden.
