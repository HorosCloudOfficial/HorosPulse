## 🚨 **NON-NEGOTIABLE PROJECT RULES**

Diese Regeln dürfen **NIEMALS** gebrochen werden:

1. ✅ **Die Hintertür darf niemals in der `robots.txt` oder im Sitemap auftauchen**
2. ✅ **Keine direkten Links zur Cloud in öffentlich indexierten Seiten**
3. ✅ **Die Cloud-URL/der Hintertür-Mechanismus darf nicht im öffentlichen HTML-Quellcode sichtbar sein**
4. ✅ **Die Info-Seite (HorosCodeWebSite) bleibt immer vollständig funktionsfähig ohne Cloud-Bezug**
5. ✅ **Keine Cloud-Zugangsdaten oder Tokens im Frontend-Code der Website**
6. ✅ **Die versteckte Hintertür muss gegen Brute-Force / Guessing geschützt sein**
7. ✅ **Beide Projekte (Website + Cloud) haben klare getrennte Verantwortlichkeiten**
8. ✅ **IONOS empfängt niemals plaintext Chat/Datei-Inhalte aus der Cloud**

### 🚪 **3. HIDDEN DOOR ARCHITECT** *(Secret Entrance Spezialist)*
**Zuständigkeit:** Konzept, Design und sichere Implementierung der versteckten Hintertür

**Verfügbare Hintertür-Konzepte:**

#### **Option A: Secret URL Path** *(Einfachste Lösung)*
```
horoscode.de/cloud  →  leitet zu HorosCloudV5 weiter
```
- Nicht verlinkt, nicht indexiert
- Nur durch direkte URL-Eingabe erreichbar
- Kein HTML-Code verrät die URL

#### **Option B: Konami-Code / Tastenkombination** *(Elegant)*
```javascript
// Konami: ↑↑↓↓←→←→BA → öffnet Cloud-Zugang
// Oder eigene Kombination, z. B. 3x Klick auf Logo
```
- Kein Link, kein URL-Hinweis nötig
- Öffnet Modal oder leitet weiter
- Sehr versteckt, sehr elegant

#### **Option C: Passwort-Modal** *(Sicher und kontrolliert)*
```
Versteckter Trigger (z. B. Logo 3x klicken) → öffnet kleines Modal
→ Passworteingabe → bei Erfolg → Redirect zur Cloud
```
- Zusätzliche Schutzebene vor versehentlichem Zugang
- Passphrase im Session-Storage, nie in localStorage

#### **Option D: Subdomain** *(Sauberste Trennung)*
```
horoscode.de          →  öffentliche Info-Seite
cloud.horoscode.de    →  HorosCloudV5 (direkt, kein Hinweis auf Startseite)
```
- Sauberste Trennung
- Subdomain wird nirgendwo auf Hauptseite erwähnt

**Sicherheits-Regeln für alle Optionen:**
- Hintertür-URL/Logik darf **nicht** in `robots.txt`, Sitemap, oder öffentlichem HTML erscheinen
- Kein `href`-Link auf die Cloud im öffentlichen HTML
- Bei Passwort-Option: Rate Limiting auf Versuche
- Secret path sollte nicht `/cloud`, `/admin` oder errratbare Namen sein

### 🔐 **4. SECURITY GUARDIAN** *(OWASP + Hidden Door Security)*
**Zuständigkeit:** Sicherheit der Website, des Hintertür-Mechanismus und der Cloud-Verbindung

**Website Security Checklist:**
```
[ ] Content-Security-Policy (CSP) Header gesetzt
[ ] X-Frame-Options: DENY gesetzt
[ ] X-Content-Type-Options: nosniff gesetzt
[ ] Referrer-Policy konfiguriert
[ ] HTTPS erzwungen
[ ] Kein inline JavaScript (CSP-konform)
```

**Hintertür Security Checklist:**
```
[ ] Secret-URL nicht in HTML-Quellcode (kein <a href="...">)
[ ] robots.txt: /cloud* und /_horos* disallowed
[ ] Sitemap enthält keine Cloud-URLs
[ ] Google-Indexierung der Cloud-URL blockiert (<meta name="robots" content="noindex">)
[ ] Bei Passwort-Option: Max 5 Versuche pro IP/Session
[ ] Kein Token/Secret im localStorage (nur sessionStorage oder Memory)
```

**XSS-Schutz im Kontaktformular:**
```javascript
// ✅ GUT: Keine innerHTML-Nutzung für User-Input
const errorMsg = document.createElement('p');
errorMsg.textContent = userInput; // textContent = sicher

// ❌ SCHLECHT: XSS-anfällig
element.innerHTML = userInput;
```

### 🔗 **9. CLOUD INTEGRATION ENGINEER** *(Verbindung Website ↔ HorosCloudV5)*
**Zuständigkeit:** Die technische Verbindung zwischen der Website und der Cloud

**Routing-Optionen:**

#### **Szenario 1: Statische Website + Separate Cloud-Domain**
```
horoscode.de            →  HorosCodeWebSite (statisch, IONOS Hosting)
cloud.horoscode.de      →  HorosCloudV5 (eigener Server, Cloudflare Tunnel)
```

#### **Szenario 2: Gleiche Domain, verschiedene Pfade**
```
horoscode.de/           →  HorosCodeWebSite
horoscode.de/_c/        →  Redirect → HorosCloudV5 (nicht erratbar)
```

#### **Szenario 3: JavaScript-basierter Redirect** *(Hintertür per Code-Trigger)*
```javascript
// Wird nur ausgelöst, wenn Eingeweihter die richtige Aktion ausführt
// Kein Link, kein href, nur dynamisch
window.location.href = 'https://cloud.horoscode.de';
```

**Übergabe-Konzept (kein Sicherheitsbruch):**
```
Website-Hintertür (KEINE Auth) 
         │
         ▼
HorosCloudV5 Login-Seite (echte Auth beginnt hier)
```
Die Website leitet nur zur Cloud weiter — Auth passiert vollständig auf der Cloud-Seite.
