## amplitude-experiment-implementation.agent

> Quelle: `.github/agents/amplitude-experiment-implementation.agent.md`  
> Typ: Agent (Experiment-Implementierung mit Amplitude MCP)  
> Status: Bedingt nutzbar (Amplitude-Zugang nötig)

### Zweck
Unterstützt die Umsetzung von Feature-Experimenten anhand von Issue-Anforderungen inklusive Instrumentierung und Variantenlogik.

### Schwerpunkte
- Anforderungen aus Issues extrahieren
- Feature + Experiment zusammen planen
- Varianten korrekt einbinden (Control/Treatment)
- Ergebnis und Experiment-Referenz dokumentieren

### Aktivierung
`/agent amplitude-experiment-implementation`

### Parameter
Keine Slash-Subparameter; Ablauf ist als Schrittfolge im Agenttext beschrieben.

### Einsatz
Universell für datengetriebene Feature-Rollouts, wenn Amplitude/MCP verfügbar ist.
