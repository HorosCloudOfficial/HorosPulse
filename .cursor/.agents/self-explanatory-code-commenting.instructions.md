## Kurzbeschreibung
Regelwerk für selbst-erklärenden Code: kommentiere primär **WARUM**, nicht **WAS**.

## Kernprinzip
**Code soll sich selbst erklären. Kommentare nur bei echtem Mehrwert.**

## Was vermieden werden soll
- Offensichtliche/redundante Kommentare
- Veraltete Kommentare
- Auskommentierter Altcodeschrott
- Deko-Kommentare als Trenner
- Changelog-Historie in Kommentaren

## Sinnvolle Kommentararten
- Komplexe Business-Logik
- Nicht-offensichtliche Algorithmen
- Regex-Erklärungen
- Externe API-Limits/Gotchas
- Klare Annotations: TODO, FIXME, HACK, NOTE, WARNING, PERF, SECURITY

## Aktivierung
Automatisch global via `applyTo: **`.

## Parameter
Keine Slash-Parameter.

## Eignung
Sehr gut als Qualitätsleitplanke für alle Sprachen im Repo.
